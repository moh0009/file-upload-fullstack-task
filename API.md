# API Reference

Base URL: `http://localhost:8080/api`

All responses use `Content-Type: application/json`. Errors follow the shape:
```json
{ "error": "human-readable message" }
```

## File Upload

### `POST /api/upload`

Upload a single chunk of a file. Chunks must be sent in order; the last chunk triggers server-side merge.

**Request** â€” `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `file` | File | âś“ | The chunk blob |
| `fileName` | string | âś“ | Original file name (must end in `.csv`) |
| `fileId` | string | âś“ | Unique session ID generated by the frontend |
| `chunkIndex` | integer | âś“ | Zero-based index of this chunk |
| `totalChunks` | integer | âś“ | Total number of chunks for this file |

**Success `200`**
```json
{ "message": "chunk uploaded" }
```

**Error responses**

| Status | Condition |
|--------|-----------|
| `400` | Missing `file` field, invalid metadata, non-CSV extension, file exceeds size limit, invalid MIME type |
| `500` | Failed to save chunk to disk or merge chunks |

### `POST /api/process`

Enqueue an uploaded file for background CSV processing.

**Request body** â€” `application/json`

```json
{
  "fileName": "grades_batch_1.csv",
  "fileId":   "grades_batch_1.csv-1713600000000-a1b2c3d",
  "userId":   ""
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `fileName` | string | âś“ | File name (must already exist in uploads dir) |
| `fileId` | string | âś“ | Same session ID used during upload |
| `userId` | string | â€” | Optional; `"premium"` â†’ higher priority, `"free"` â†’ lower priority |

**Success `202`**
```json
{
  "job_id":      "grades_batch_1.csv-1713600000000-a1b2c3d",
  "message":     "queued for processing",
  "progress_ws": "/ws/progress?fileId=grades_batch_1.csv-1713600000000-a1b2c3d"
}
```

**Error responses**

| Status | Condition |
|--------|-----------|
| `400` | Invalid or missing `fileName` / `fileId` |
| `503` | Processing queue is full |

## WebSocket â€” Progress Stream

### `GET /api/ws/progress`

Upgrade to WebSocket. The server pushes JSON progress frames as the file is uploaded and processed.

**Query parameters**

| Param | Required | Description |
|-------|----------|-------------|
| `fileId` | âś“ | The session ID to subscribe to |

**WebSocket message format**

```json
{
  "type":   "progress",
  "job_id": "grades_batch_1.csv-1713600000000-a1b2c3d",
  "progress": {
    "stage":       "uploading | parsing | moving | complete",
    "upload_pct":  75.5,
    "process_pct": 0,
    "rows":        0
  }
}
```

| Stage | Meaning |
|-------|---------|
| `uploading` | Chunk is being received by the server |
| `parsing` | CSV is being streamed into the staging table |
| `moving` | Staging rows are being validated and moved to `students` |
| `complete` | All rows committed; file removed from disk |

**Reconnection**: The frontend uses `ManagedWebSocket` with exponential back-off (up to 5 retries, max 30 s delay). Connections are allowed from `http://localhost:3000` and `http://localhost:3001`.

## Students

### `GET /api/students`

Retrieve a page of student records.

**Query parameters**

| Param | Type | Description |
|-------|------|-------------|
| `pageSize` | integer (required) | Number of records to return (max 100 recommended) |
| `sortBy` | string | Column + direction, e.g. `name ASC`, `grade DESC` |
| `name` | string | Partial name search (SQL `LIKE`) |
| `subject` | string | Exact subject match |
| `gradeMin` | integer | Minimum grade (inclusive) |
| `gradeMax` | integer | Maximum grade (inclusive) |
| `afterId` | integer | Keyset cursor â€” ID of last record (for next page) |
| `afterValue` | string | Keyset cursor â€” sort column value of last record |
| `beforeId` | integer | Keyset cursor â€” ID of first record (for previous page) |
| `beforeValue` | string | Keyset cursor â€” sort column value of first record |

> **Keyset pagination**: On the first page omit cursor params. For subsequent pages pass either `afterId + afterValue` (next) or `beforeId + beforeValue` (previous), using the boundary row of the current page.

**Success `200`** â€” array (empty array when no results, never `null`)

```json
[
  { "id": 1, "name": "Alice Smith", "subject": "Mathematics", "grade": 87 },
  ...
]
```

**Error responses**

| Status | Condition |
|--------|-----------|
| `400` | `pageSize` is missing or not a positive integer; `gradeMin` / `gradeMax` not integers; cursor IDs not integers |
| `500` | Database query failed |

### `GET /api/students/count`

Count student records matching the current filters. Call this before `GET /api/students` to know the total pages.

**Query parameters** â€” same filter params as `GET /api/students` (no pagination / sort params)

**Success `200`**
```json

```

### `DELETE /api/students/:id`

Delete a student by ID.

**Path parameter**: `id` â€” integer

**Success `200`**
```json

```

**Error responses**

| Status | Condition |
|--------|-----------|
| `400` | `id` is not an integer |
| `404` | No student with that ID |
| `500` | Database error |
